# FDE 传播型三级顺序晋级挑战设计

## 1. 目标

将现有可以任意选择初级、中级、高级的水平测试，改为只能按“初级 → 中级 → 高级”完成的传播型能力挑战。只有完成前一级完整挑战并达到严格晋级标准，才能解锁下一级。

本站用于传播、水平预评估和自我校准，不接收真实项目材料，不作为 FDE 毕业或认证结果。未来的真实考试库和正式考务系统与本站分离。

## 2. 依据与边界

严格感来自现有 FDE 毕业标准的分数口径：70 分达标，85 分优秀。本挑战将 85 分作为等级晋级线，同时增加模块最低分要求，防止总分掩盖明显短板。

以下内容不在本次范围：

- 学员登录、班级与考务管理。
- 真实考试题库、项目材料、工具实操与答辩。
- 正式成绩档案、证书编号与认证结论。
- 服务端防作弊和身份核验。

## 3. 挑战路径

1. 所有用户从初级开始，不允许跳级。
2. 初级完整挑战包含 100 题。
3. 初级达到晋级标准后，解锁中级 60 题完整挑战。
4. 中级达到晋级标准后，解锁高级 40 题完整挑战。
5. 高级达到晋级标准后，生成“完成三级挑战 · 高阶潜质”结果。
6. 用户需在三级完整挑战中累计完成 200 道题，但允许跨会话自动保存和续答。

快速潜质测试仍可保留为入口，但结果只能提示能力特征，不能推荐跳级，也不能改变解锁状态。
快速测试原有的等级化结论改为“能力侧写”，完成后的主行动始终是“从初级开始晋级”。

## 4. 模式规则

每个已解锁等级保留两种模式：

- **完整挑战**：计入晋级，必须完成本级全部题目。每次新开始时打乱题目顺序，刷新恢复时保持本次题序。
- **随机模拟**：只用于练习，不记录等级成就，不解锁下一级，分享卡必须显示“模拟练习”。

中级和高级未解锁时，完整挑战与随机模拟都不可进入。

## 5. 评分与晋级标准

### 5.1 题目评分

- 单选题和判断题：完全正确得 1 分，其他得 0 分。
- 多选题：所有正确选项与用户选择完全一致才得 1 分；少选、多选或选入错项均得 0 分。
- 未作答得 0 分。
- 总分按本级得分除以本级题数，换算为 100 分制。

### 5.2 结果分类

- 0-69 分：未达标。
- 70-84 分：本级达标，但未达晋级线。
- 85-100 分：总分达到晋级候选线。

### 5.3 晋级判定

只有同时满足以下条件，才算本级晋级成功：

1. 当次为完整挑战。
2. 已完成本级所有题目。
3. 总分不低于 85 分。
4. 每个能力模块得分均不低于 70 分。

如总分达到 85 但某个模块低于 70，结果显示“总分达线，存在模块短板，暂不晋级”。

## 6. 界面与交互

### 6.1 选级页

- 顶部增加三级路径进度：`01 初级 → 02 中级 → 03 高级`。
- 初级默认显示“必经起点”。
- 未解锁等级降低视觉强度，显示锁图标、解锁条件和前一级最佳成绩。
- 按钮使用真实 `disabled` 状态；JavaScript 事件层再次检查权限，避免只靠样式锁定。
- 已晋级的等级显示最佳总分、最低模块分和完成时间。

### 6.2 模式页

- 完整挑战明确标注“计入晋级”。
- 随机模拟明确标注“仅练习，不解锁下一级”。
- 显示本级晋级线：总分 85，每模块不低于 70。

### 6.3 成绩页

- 完整挑战结果分为“未达标”、“本级达标”、“晋级成功”三种。
- 结果页同时显示总分线和模块底线，明确告诉用户未晋级的原因。
- 晋级成功时提供“进入下一级”主按钮；其他情况提供“重新挑战本级”。
- 模拟结果始终显示“模拟练习”，不输出晋级结论。
- 错题、解析、模块成绩、重测和分享卡继续保留。

### 6.4 多选题交互提示

- 单选题和判断题使用圆形选择控件，题型标签显示“单选题”或“判断题”。
- 多选题使用方形复选控件，题型标签使用更高对比色。
- 每道多选题的题型标签下方固定显示：“请选择全部正确答案，少选、多选、错选均不得分。”
- 当前题为多选题时，交卷确认面板再次提示严格多选计分规则。
- 题型提示使用 `role="note"` 并与当前题建立可读关联，不只依赖颜色区分。

### 6.5 品牌与传播边界

首页、选级页、工作台、成绩页和分享卡继续保留 OneX 体系与版权文字，并新增统一边界：

> 本结果来自传播型 FDE 能力挑战，不代表正式毕业、认证或真实项目能力结论。

## 7. 本地状态与迁移

新增独立的版本化晋级状态，保存以下数据：

- 每个等级完整挑战的最佳总分。
- 当次最佳成绩对应的模块分数。
- 是否满足晋级条件。
- 完成时间。

解锁状态由成绩证据实时推导，不单独保存一个可脱离成绩修改的 `unlocked=true`。

上线前的旧版站点允许自由选级，因此旧的中高级进度和成绩不能当作晋级证据。迁移规则如下：

- 新版首次访问默认只解锁初级。
- 旧的中级、高级活跃考试不恢复，并提示规则已升级。
- 旧的初级未完成进度可续答，但晋级必须使用新评分规则完成交卷。
- 用户可一键清空所有挑战状态并从初级重新开始。

浏览器禁用本地存储时，允许完成当前页面考试，但不能在刷新后恢复，也不能在新会话中保留解锁状态。

## 8. 最终姓名分享卡

只有完成高级完整挑战且满足晋级标准时，才显示最终姓名分享区。

- 姓名或昵称为可选项，最长 20 个字符。
- 输入值去除首尾空格、换行和控制字符；空值显示“匿名挑战者”。
- 姓名只保留在当前页面内存中，不写入本地存储，不上传分析服务，不进入服务器日志参数。
- 最终分享卡显示挑战者姓名、三级最佳总分、累计完成 200 题、最终挑战结论和非认证边界。
- 下载文件名可使用经过清理的姓名；不得包含路径分隔符或其他非法文件名字符。
- 初级、中级和模拟结果卡不显示“最终挑战者”姓名区，避免在未完成全部挑战前形成高级结论。

## 9. 匿名浏览与转化统计

### 9.1 定位

在同一阿里云服务器上部署轻量统计服务，向站点运营者提供密码保护的私有数据看板。本次不在公开页面展示访问数，避免早期样本过小或公开计数被刷量影响传播判断。

### 9.2 事件范围

前端只发送以下允许事件：

- `page_view`：页面访问。
- `quick_start` / `quick_complete`：快速潜质测试开始和完成。
- `level_start` / `level_complete`：按等级和模式记录开始与完成。
- `level_unlock`：解锁中级或高级。
- `final_complete`：完成全部三级晋级挑战。
- `share_generate`：生成普通或最终分享卡。

`level_complete` 可携带等级、模式、整数总分、是否晋级和最低模块分，但不得携带任何单题答案、错题内容或姓名。

### 9.3 匿名识别与隐私

- 首次访问时在浏览器本地生成随机、无业务含义的访客 ID，用于统计 UV。
- 每个浏览器会话生成独立会话 ID，用于统计会话和漏斗。
- 服务不存储姓名、答案、原始 IP 地址或完整 User-Agent。
- 只存储来源主机名、设备大类、允许事件字段和服务端时间。
- 浏览器开启 `Do Not Track` 时不发送统计事件。
- 自动化浏览器环境 `navigator.webdriver === true` 时不发送事件，避免测试污染线上数据。
- 前端在页脚提示使用匿名、无 Cookie 的运营统计；用户清除本地浏览器数据后，访客 ID 同步消失。

### 9.4 服务与存储

- 使用轻量 Python 服务和 SQLite，只监听 `127.0.0.1`，由 Nginx 反向代理。
- 数据库位于 `/var/lib/fde-analytics/analytics.db`，站点重新部署时不覆盖。
- 事件接口只接受白名单字段、限制请求体大小，并由 Nginx 限速。
- 事件接口失败、超时或服务不可用时，前端静默放弃，不影响站点、答题、进度保存或分享卡。
- 原始匿名事件保留 180 天；每日聚合数据长期保留。清理任务不得阻塞事件写入。

### 9.5 私有数据看板

- 看板位于 `https://fde.onex.plus/stats/`，由 Nginx Basic Auth 保护。
- 管理员用户名与强随机密码在服务器部署时生成，密码哈希存在服务器外置文件，明文密码不进 Git。
- 看板支持今日、7 天、30 天和全部时间范围。
- 核心指标包含 PV、UV、会话数、来源主机、设备大类、每日趋势、快速测试漏斗、三级开始/完成/解锁漏斗、分数分布和分享卡生成次数。
- 看板不提供任何单个用户的可识别行为详情。

## 10. 核心模块

- `assessment-levels.js`：等级顺序、达标线、晋级线和模块底线。
- `exam-scoring.js`：严格题目评分、成绩分类和晋级判定。
- `exam-progression.js`：纯函数形式的成绩证据、解锁判定、更新最佳成绩与版本化存储。
- `exam-state.js`：单次考试的题序、答案、当前题和恢复校验。
- `exam-app.js`：三级 UI 状态、访问守卫、交卷后更新证据和下一级导航。
- `exam-share-card.js`：根据模式和晋级结果生成边界清晰的分享卡。
- `analytics.js`：前端匿名访客/会话标识、事件白名单和非阻塞上报。
- `analytics/server.py`：SQLite 事件写入、聚合查询和看板数据 API。
- `stats/`：密码保护的私有数据看板页面。
- `deploy/fde-analytics.service`：统计服务的 systemd 单元。

## 11. 异常与安全边界

- 直接触发未解锁等级的 DOM 事件时，拒绝进入并回到选级页。
- 恢复未解锁等级的旧活跃进度时，删除活跃指针并显示规则升级提示。
- 晋级存储解析失败或版本不匹配时，不解锁任何高等级，允许用户清空重开。
- 客户端存储可被本机用户修改，因此所有结果都必须保留“非正式认证”边界，不可用于真实毕业判定。
- 统计服务不信任客户端事件和分数；数据只用于运营趋势，不用于真实考务。
- 数据看板未通过 Basic Auth 时不返回 HTML 或聚合数据。

## 12. 测试与验收

### 12.1 单元与纯函数测试

- 多选少选、多选、含错项均得 0 分。
- 70 分为达标线，85 分为总分晋级候选线。
- 总分 85 且全模块 70 以上时晋级成功。
- 模块存在 69 分时，即使总分 90 也不晋级。
- 模拟成绩不写入晋级证据。
- 中级仅在初级证据合格后可访问，高级仅在中级证据合格后可访问。
- 较差的重考成绩不覆盖已存在的最佳成绩证据。
- 完整挑战新开始会打乱题序，恢复时题序不变。
- 姓名清理会截断至 20 字符，删除控制字符，并生成安全文件名。
- 统计服务拒绝未知事件、额外字段、超长 ID 和超限请求体。
- 聚合 API 能正确计算 PV、UV、会话、漏斗和按日趋势，且不返回原始事件记录。

### 12.2 浏览器验收

- 新用户只能进入初级，中级和高级均为禁用状态。
- 快速潜质测试即使满分，也不解锁中级或高级。
- 初级模拟满分也不解锁中级。
- 初级完整挑战达到晋级标准后，中级立即解锁。
- 中级完整挑战达到晋级标准后，高级立即解锁。
- 刷新后解锁状态保留。
- 旧版未解锁等级的活跃进度不能绕过守卫。
- 成绩页、分享卡和所有等级页均显示传播型挑战边界。
- 多选题明确显示“请选择全部正确答案，少选、多选、错选均不得分”，并使用方形复选控件。
- 只有最终高级晋级成功页显示姓名输入，分享卡显示输入姓名与三级成绩。
- 姓名不出现在任何统计请求中。
- 本地、DNT 和自动化浏览器环境不发送统计事件。
- 桌面与手机宽度无水平溢出，浏览器控制台无错误。

### 12.3 公网验收

- GitHub `main` 包含最新规则和部署脚本。
- 阿里云从 GitHub `main` 部署，站点文件哈希与本地一致。
- `https://fde.onex.plus/` 返回 HTTP 200，公网完成上述关键路径回归。
- 匿名事件经 Nginx 写入 SQLite，事件失败不影响站点主流程。
- `https://fde.onex.plus/stats/` 未认证返回 401，正确认证后返回 200 且看板可用。
- 数据库位于站点发布目录外，重新部署后历史统计保留。

## 13. 验收结论

完成后，站点应让用户清晰感知：高级潜质不能自选，必须用前两级完整成绩换取；分数不能只靠模块偏科或多选少选获得；多选规则在作答前已明确告知；只有完成全部三级的人可以将姓名写入最终分享卡；运营者可在不收集答案和姓名的前提下看到传播与转化数据；所有分享结果都能传播，但不伪装成正式毕业或认证。
