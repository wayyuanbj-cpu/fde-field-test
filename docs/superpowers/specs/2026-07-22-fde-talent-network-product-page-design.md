# OneX FDE 人才网络产品页设计

## 1. 目标与定位

本轮把现有 `/talents/` 从基础公开目录升级为“企业找 FDE”优先的人才网络产品页。页面的第一任务是让企业项目负责人快速理解：这里有哪些 FDE、分别能解决什么企业问题、当前认证与交付状态是什么、适合怎样的合作方式。

人才不能被呈现为普通简历。每一张人才卡都是一份经过授权和脱敏处理的“交付能力产品说明”，由能力定位、服务边界、标准服务包、项目证据、合作方式和当前档期共同构成。

第二任务是承接已有工程师申请入库。页面保留“我是 FDE，申请入库”入口，但不让工程师招募信息压过企业找人的主路径。

## 2. 用户与核心任务

### 2.1 企业项目负责人

进入页面后应在首屏完成三件事：确认 OneX FDE 人才网络的作用、理解人才状态差异、开始按项目条件筛选人才。

### 2.2 已有交付经验的 FDE

能够看到公开档案的质量标准，并进入独立的入库申请路径。公开测试成绩不能自动转化为人才信用，入库和认证仍是独立流程。

### 2.3 正在成长为 FDE 的用户

能够从辅助入口进入 FDE 潜质测试或小班培训，但不作为本页首屏的主要转化目标。

## 3. 页面信息架构

页面按以下顺序组织：

1. 全局导航：返回 FDE 测试站、查看成长路径、申请入库。
2. 企业找人首屏：一句话定义人才网络，突出“按交付能力找 FDE”，提供“浏览人才”和“提交项目需求”两个动作。
3. 人才状态说明：用四种清晰状态区分人才库成员、认证审核中、OneX 认证 FDE、OneX 交付 FDE。
4. 项目条件筛选：成员状态、城市、能力标签、可用状态四类条件；桌面端保持一行，移动端改为两列或单列。
5. 人才产品卡片：展示能力定位、服务方式、档期、能力标签和关键证据，并进入该工程师的独立页面。
6. 工程师独立页面：完整展示能力概要、标准服务包、脱敏项目证据、不适用范围、认证状态和合作入口。
7. 认证边界：固定说明人才库展示、正式认证、认证徽章和真实交付记录之间的差异。
8. 双向入口：企业提交项目需求；FDE 申请入库。
9. 页脚：隐私字段边界和 OneX 社区归属。

## 4. 首屏设计

主标题采用“按交付能力，找到合适的 FDE。”，避免“人才市场”“专家平台”等泛化表述。辅助文案明确 OneX 展示的是可验证能力、服务边界和合作方式，而不是考试分数排行榜。

首屏右侧使用“项目匹配控制台”作为页面的标志性视觉：以真实筛选维度组合出一份项目需求摘要，例如行业场景、问题类型、合作方式和项目阶段。它不是装饰性的仪表盘，而是帮助企业理解后续筛选逻辑的结构化预览。

首屏不使用虚构的工程师数量、企业数量、成功率、剩余名额或合作金额。首批真实人才数据未发布前，不显示未经证实的业务统计。

## 5. 人才状态与徽章边界

| 状态 | 页面显示 | 徽章规则 | 对企业的含义 |
|---|---|---|---|
| 人才库成员 | 人才库成员 | 不显示认证徽章 | 资料已授权并审核发布，尚未完成 OneX 认证 |
| 认证审核中 | 认证审核中 | 不显示认证徽章 | 已进入认证流程，不代表已经通过 |
| OneX 认证 FDE | OneX 认证 FDE | 可显示认证等级、编号或有效期 | 已完成规定的综合认证流程 |
| OneX 交付 FDE | OneX 交付 FDE | 可显示交付记录型标识 | 存在经过授权和核验的项目交付记录 |

页面不得把公开测试分、入库测试分或培训结业成绩当作认证徽章。未认证档案始终显示“尚未完成 OneX 认证”的明确提示。

## 6. 人才卡片

人才卡片首屏信息控制在以下层级：

- 档案编号与人才状态；
- 能力定位和一句话标题；
- 城市、服务方式、当前档期；
- 最多 6 个核心能力标签；
- 能力概要；
- 标准服务包；
- 脱敏项目证据；
- 明确的不适用范围。

卡片不展示真实姓名、手机号、邮箱、微信、身份证明、客户机密、未脱敏项目材料或精确考试分数。公开档案使用经本人授权的展示名；测试与演示只能使用明确标注的虚构资料。

卡片的主要动作是“查看独立主页”，进入该工程师唯一、可分享的公开 URL。卡片不提供公开私信或直接联系方式。企业合作先进入统一的项目需求入口，由 OneX 运营进行场景确认和人工匹配。

## 7. 工程师独立页面

每位已发布工程师都有唯一公开地址：`/talents/<slug>/`。`slug` 来自已审核的人才档案，只允许小写字母、数字和连字符；修改展示名不会改变公开地址。

独立页面按以下结构组织：

1. 返回人才目录与当前人才状态；
2. 展示名、能力定位、城市、服务方式、档期和能力标签；
3. “我能解决的问题”：使用公开档案中的能力概要；
4. “标准服务包”：说明适合的项目范围、交付内容和合作方式；
5. “可核验的脱敏证据”：只显示已授权的证据摘要；
6. “不适合的项目”：主动说明能力边界，避免企业误配；
7. “认证与交付状态”：明确区分人才库成员、认证 FDE 和交付 FDE；
8. “带着这位 FDE 提交需求”：将人才 `slug` 作为非敏感匹配线索带入统一企业需求入口；
9. 公开信息与隐私边界说明。

未发布、撤回授权或不存在的 `slug` 返回找不到档案的页面状态，不显示缓存档案或相似人才猜测。人才网络灰度开关关闭时，独立页面与目录保持相同的关闭口径。

独立页面首期不展示照片、客户 Logo、客户原文评价或可直接联系的个人信息，除非后续数据合同增加对应字段、授权记录和撤回机制。

## 8. 数据与接口

本轮继续复用现有同源接口：

- `GET /api/network/config`：读取人才网络和公开目录灰度开关。
- `GET /api/network/public/talents`：读取经过服务端字段投影的公开档案。
- `GET /api/network/public/talents/<slug>`：读取单个已授权、已发布工程师的公开档案。

筛选参数保持 `status`、`city`、`tag`、`availability`。浏览器只渲染接口允许返回的公开字段，不从其他接口拼接身份信息。

人才网络开关或公开目录开关关闭时，页面进入“灰度准备中”状态，并提供 FDE 测试与小班培训入口；不能退回首页内容，也不能伪造人才卡片。接口失败时保留当前筛选条件，显示明确错误和重试按钮。

目录页和独立页面使用同一公开字段集合，独立页面不能通过额外接口获取真实姓名或内部材料。独立页面采用静态 HTML 外壳和同源详情 API；Nginx 与本地验收服务器将 `/talents/<slug>/` 映射到详情页外壳，由浏览器根据路径读取对应档案。

## 9. 视觉系统

页面与 FDE 测试页和培训页保持同一 UI 家族：

- 夜航蓝 `#061427`：页面主背景；
- 深层蓝 `#0B1E37`：卡片与筛选面板；
- 电场蓝 `#3F67FF`：主要动作、状态轨迹和键盘焦点；
- 冰白 `#F2F6FF`：主文字；
- 雾蓝 `#9AAECA`：说明文字；
- 警示橙 `#FF7248`：未认证边界和风险提示。

标题使用高密度无衬线中文字体栈，数据标签、档案编号和筛选状态使用等宽字体。页面减少圆角和渐变，以精确分隔线、字段标签和结构化排版建立“工程交付控制台”的感觉。

页面唯一的视觉记忆点是首屏“项目匹配控制台”：企业选择的是问题、约束和交付阶段，再由这些条件指向合适的 FDE，而不是先看人物头衔。

独立页面沿用同一视觉系统，但把“能力定位”作为页面主视觉，不做通用个人简历版式。页面使用一条从“企业问题”到“服务包”再到“证据”的纵向交付轨迹，让企业先理解能解决什么，再理解是谁。

## 10. 响应式与无障碍

- 桌面端内容宽度不超过 1240px，首屏为两栏，人才卡片为两列。
- 390px 移动端不出现横向滚动；首屏、状态说明、筛选和人才卡片全部改单列。
- 所有筛选项有可访问名称，状态变化使用 `aria-live`。
- 键盘可以依次访问导航、主要动作、筛选项、重试和双向入口。
- 尊重 `prefers-reduced-motion`，不使用必须依赖动画才能理解的状态。
- 主文字、辅助文字、链接、输入框和焦点态达到 WCAG AA 对比度。

## 11. 测试与验收

### 11.1 确定性测试

验证首屏主张、四种人才状态、筛选字段、人才卡片详情链接、独立 URL 解析、认证边界、企业需求入口、FDE 入库入口以及隐私字段禁用规则。

### 11.2 浏览器验收

使用固定虚构档案验证：

- 1440px 桌面首屏与两列人才卡片；
- 390×844 移动端无横向滚动；
- 城市和能力标签筛选写入 URL；
- 从人才卡片进入 `/talents/<slug>/`，并显示与详情 API 一致的公开档案；
- 独立页面的企业需求入口携带正确人才 `slug`；
- 无效、未发布和灰度关闭的独立页面显示受控状态；
- 未认证档案不出现认证徽章；
- 目录关闭、空结果、接口失败和重试状态；
- 控制台无错误，键盘焦点可见。

### 11.3 回归验收

原有中文测试、英文测试、FDE 指南、培训招生页、统计后台、匿名分析以及公开测试与认证边界不能回退。

## 12. 本轮范围

本轮实现人才网络产品页、公开目录体验升级和每位已发布工程师的独立公开页面，不实现自动推荐算法、公开联系方式、在线聊天、企业需求写入、人才个人账号、入库考试、认证评分或真实人才批量发布。这些能力继续由后续独立计划承接。
