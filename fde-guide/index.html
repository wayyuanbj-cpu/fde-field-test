FDE REFERENCE / ANSWER-FIRST GUIDE

FDE 是什么？
不只是会用 AI。

Field Deployment Engineer（FDE）是连接企业问题、AI 能力、运营约束与可验收交付结果的复合型角色。他不是只做演示的售前，也不是只接需求写代码的工程师。

01 / DEFINITION

WHAT IS AN FDE?

FDE 的核心，是对“从问题到结果”负责。

FDE 把模糊的企业诉求转化为可执行的 AI 场景，在真实数据、权限、流程和组织约束下完成试点、上线、验收与持续使用。

为什么这个角色现在重要？

AI 能力更容易获取了，但企业交付的难点没有消失：问题不清楚、数据不可用、异常无人接管、价值无法量化、组织不愿改流程。FDE 的价值是把这些现实条件纳入方案，而不是展示一个脱离现场的 Demo。

02 / RESPONSIBILITY

WHAT DOES AN FDE DO?

FDE 通常负责六件事。

诊断真问题、选择场景、设计方案、守住风险边界、推进交付，并用证据证明产品已经产生可持续的业务结果。

01 / DIAGNOSE

业务诊断

访谈关键人、还原真实流程，找出痛点、频率、成本与现有证据。

02 / PRIORITIZE

场景取舍

同时考虑价值、数据就绪度、风险、复杂度和推广阻力，选第一个值得做的场景。

03 / DESIGN

方案与工作流

明确输入、模型、知识、工具、权限、人工审批、异常处理和输出。

04 / PROVE

交付与验收

用基线、试点、质量门槛、用户采用率和业务指标完成可追溯的验收。

03 / CAPABILITY

OneX 用五个维度观察 FDE 能力。

真正的 FDE 不是某一项工具特别熟练，而是能同时完成业务诊断、场景判断、方案构建、风险控制和推进交付。

05 / DEL

推进交付

跨过 Demo，进入试点、上线、验收和持续使用。

04 / COMPARISON

ROLE BOUNDARIES

FDE 与售前、产品经理、软件工程师有什么不同？

关键不是“会不会写代码”，而是责任边界：FDE 对企业现场中方案能否完成交付并产生可验收结果负责。

角色主要目标典型产出对现场结果的责任
FDE把 AI 能力变成可运行、可验收的业务结果诊断、试点、工作流、控制面、验收证据跨越问题定义、技术、交付和采用
售前解决方案证明产品与客户需求的匹配演示、方案书、技术澄清通常聚焦签约前或交接前
产品经理决定通用产品应解决什么问题路线图、需求、产品指标对产品方向负责，不一定驻场交付
软件 / ML 工程师构建可靠的软件、数据或模型系统代码、模型、服务、基础设施对技术质量负责，责任范围由团队设计决定
05 / ASSESSMENT

HOW THE TEST WORKS

OneX FDE 公开水平测试怎么评？

先用 12 道企业情境题判断能力倾向，再按初级 100 题、中级 60 题、高级 40 题的顺序挑战，验证知识、情境判断与复杂交付决策。

QUICK / 12

测测 FDE 灵根

约 8 分钟，输出五维能力侧写；只判断潜质，不用于晋级。

L1 / 100

初级·理论基础

模型、Prompt、RAG、Agent、场景设计、交付与治理。

L2 / 60

中级·情境交付

在真实项目压力中评估诊断、取舍、方案、推进和验收。

L3 / 40

高级·复杂决策

复杂系统、跨部门治理、规模化交付、业务价值和风险兜底。

STRICT PROGRESSION STANDARD

所有人必须从初级开始，不得跳级。完整挑战总分至少 85，且每个模块至少 70，才解锁下一级。随机模拟只用于练习，不会解锁等级。

06 / BOUNDARY

WHAT THIS RESULT MEANS

线上高分，不等同于已经是 FDE。

这个网站是面向公众的 FDE 能力挑战和传播工具，用来观察理论理解与情境判断，不等同于正式毕业、认证或真实项目能力结论。

正式评估还需要什么？

真实项目材料、可运行方案、验收证据、现场质询、数据与权限边界、客户或业务方反馈，以及对长期使用结果的复盘。线上成绩可以成为一个输入，但不能替代这些证据。
